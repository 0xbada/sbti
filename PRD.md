**작성일**: 2026-04-22

## 1. 제품 정의

SBTI(Silly/Satirical Behavioral Type Indicator)는 2026년 4월 중국발 바이럴 MBTI 패러디 테스트의 **한국어 로컬라이즈 독립 사이트**. 30문항 퀴즈로 15개 차원을 측정해 27개 풍자적 성격 유형 중 하나로 매칭하고, 카카오톡·인스타그램 공유를 통한 바이럴 트래픽을 목표로 한다.

## 2. 배경 · 기회

- 원작은 중국 영상 크리에이터 '아푸'가 친구의 금주 설득을 위해 만든 패러디 테스트 — 2026년 4월 중국·한국 SNS에서 동시 바이럴
- 한국 검색량: **월 56,000+ "SBTI" 관련 쿼리** (2026년 4월 기준)
- 경쟁 사이트 이미 다수 선점 — `sbti-test.com`, `sbti-test.net`, `sbti.global`, `sbti.digital`, `sbtipersonality.org`, `nano-banana.com/ko/sbti` 등
- 선점 사이트 대부분이 **번역체 한국어**, **카카오 공유 미최적화**, **한국 맥락 밈 부재** — 후발 차별화 여지 있음

## 3. 타겟 사용자

- **20~35세 SNS 헤비 유저** (카카오톡 단톡방, 인스타 스토리, X 공유)
- MBTI에 피로감 느끼는 유저, 자조·팩폭형 유머 선호층
- 모임·회식에서 밈 소재 찾는 그룹 주도자

핵심 행동: **테스트 완료 → 결과 스크린샷/링크 공유 → 지인 유입**

## 6. 기능 요구사항

### 6.1 퀴즈 플로우
1. 랜딩 → "테스트 시작" 버튼
2. 문항 화면 (1/30 진행률 표시) — 각 문항마다 3~5개 선택지 (L/M/H 평점형)
3. 답변 선택 즉시 다음 문항 자동 전환 (이전 버튼 제공)
4. 30문항 + 조건부 히든 문항 완료 시 채점 → 결과 라우팅
5. 결과 페이지: 유형 코드·한국어명·한줄 설명·차원 시각화(15차원 그래프)·공유 버튼

### 6.2 채점 알고리즘
- 입력: 30개 답변 각각 L/M/H 3단계
- 차원 점수: 각 문항이 1~2개 차원에 기여 → 15차원 벡터 생성
- 유형 매칭: 27개 유형의 "이상적 차원 프로필"과 유클리드/코사인 거리 계산 → 최근접 유형 선택
- 매칭률 60% 미만 시 HHHH 폴백

### 6.3 공유 기능
- **카카오톡 링크 공유** (Kakao JavaScript SDK) — 유형별 썸네일·메시지 프리셋
- **Open Graph 메타태그** (`og:image`, `og:title`, `og:description`) — 유형 결과 페이지마다 차별화
- **결과 이미지 다운로드** — `html2canvas` 등으로 결과 카드 캡처
- **링크 복사 버튼** — `navigator.clipboard`

## 14. 열린 결정 (구현 전 합의 필수)

각 항목 결정 없이는 개발 착수 불가. 결정 시 이 문서에 이력 기록.

### 14.1 도메인 전략 ✅ 확정 (2026-04-22)
**`sbti.funhada.xyz`** — funhada.xyz 서브도메인. Cloudflare Pages 커스텀 도메인으로 바인딩.

**Downstream 영향:**
- funhada.xyz 루트·형제 서브도메인에 애드센스 연동된 사이트가 있다면 SBTI의 수위 있는 유형명이 해당 사이트의 site-level 정책 평가에 영향 가능 — **funhada.xyz 현재 용도 확인 필요**
- `.xyz` TLD는 한국 시장에서 `.com`/`.kr`보다 클릭률·신뢰도 낮은 편 → 초기 유입은 SEO보다 **카카오톡/SNS 직접 공유 중심**으로 설계
- 서브도메인 구조상 루트 도메인의 검색 권위(domain authority) 공유 불가 — SBTI 자체 백링크 필요

### 14.2 채점 알고리즘 ✅ 확정 (2026-04-22)
**15차원 벡터 최근접 매칭** — 기본 거리 함수는 **유클리드 거리** (L2 norm).

**알고리즘:**
1. 30개 답변 각각 L/M/H를 숫자로 매핑 (예: L=1, M=2, H=3)
2. 문항별 기여 차원 가중치를 `questions.json`에 선언 → 15차원 벡터 생성
3. 각 차원 점수 정규화 (0~1 범위)
4. 27개 유형의 이상적 차원 프로필(`types.json`의 `dimensionProfile`)과 유클리드 거리 계산
5. 최소 거리 유형 선택 + 매칭률(`1 - distance/maxDistance`) 계산
6. 매칭률 < 60% → HHHH 폴백 유형
7. 특정 답변 조합 충족 시 DRUNK 히든 유형으로 오버라이드 (14.6 결정에 따라)

**설계 원칙:**
- 거리 함수는 `src/js/scoring.js`의 `computeDistance(vec1, vec2)` 함수로 추상화 → 유클리드 ↔ 코사인 1줄 교체 가능
- 유형 프로필(`dimensionProfile`)은 데이터로 분리해 알고리즘 변경 없이 튜닝 가능
- 스냅샷 테스트: 대표 답변 세트 10개 → 기대 유형 매핑 저장, 리팩토링 시 회귀 방지

**유클리드 선택 근거**: L/M/H 3단계 평점은 벡터 magnitude 차이가 작아 유클리드가 직관적·해석 가능. 코사인은 방향만 비교해 "모든 차원 낮음 프로필"을 "모든 차원 높음 프로필"과 가깝게 판정하는 역효과 가능. 15차원은 dense한 편이라 유클리드가 안정적.

**MVP 무광고 론칭** — 트래픽·공유율 검증 후 재평가.

**결정 근거:**

**재평가 트리거 (론칭 후):**

### 14.5 AI 초상화 기능 ✅ 확정 (2026-04-22)
**P1에 포함** — 결과 페이지의 선택적 보너스 기능.

**중요 구분: 공유 프리뷰 vs AI 초상화**
유저 의도("공유 시 웃긴/재밌는 이미지 카드로 CTR 상승")를 정확히 구현하려면 두 기능을 분리해야 함 — 링크 공유의 OG 프리뷰는 기술적으로 **사전 생성된 고정 URL 이미지**여야 하므로 AI 초상화가 담당 불가.

| 기능 | 역할 | 우선순위 | 구현 방식 |
|---|---|---|---|
| **27개 정적 타입 OG 카드** | 카카오/SNS 링크 공유 프리뷰 — CTR 결정 요인 | **P0 (필수)** | 디자이너 고퀄 제작, 유형별 고정 `og:image` |
| **AI 초상화 생성** | 결과 페이지 인앱 추가 공유 훅 (본인 얼굴 합성) | **P1 (보너스)** | 결과 페이지 "내 얼굴로 카드 만들기" 버튼 |

결론: OG 카드 품질이 링크 클릭률을 좌우하므로 P0에서 절대 타협 불가. AI 초상화는 결과 본 유저의 재공유·인스타 스토리용 차별화 훅.

**AI 초상화 구현 요구사항 (P1):**
- 이미지 생성: Cloudflare Workers AI (Stable Diffusion, 무료 tier) 또는 Replicate SDXL
- 이미지 저장: Cloudflare R2 (생성물 7일 임시 저장, 원본 사진은 저장 금지)
- 업로드 보안: 클라이언트 → Worker 검증 → 모델 호출 → 결과만 반환
- 비용: 생성당 $0.01~0.05, 월 1,000건 기준 최대 $50
- 개인정보처리방침 업데이트 필수 (얼굴 사진 처리 고지)

**P1 릴리스 조건:** P0 론칭 후 MAU 5,000+ 달성 + 월 $50~$100 운영비 확보.

### 14.6 히든 문항 (DRUNK 유형) ✅ 확정 (2026-04-22)
**원작 DRUNK 히든 유형 오마주 + 한국 정서 톤 조정.**

**유지하는 것 (원작 충실):**
- 코드 `DRUNK` 그대로 (공유 일관성, §14.3 원칙)
- 히든 트리거 메커니즘 — 특정 답변 조합 충족 시에만 발현, 일반 27유형 매칭을 오버라이드
- 원작자 아푸의 본래 의도(절주 환기) 존중

**한국 정서 적응 (조롱 → 자조·공감):**
- 한국어 별명: "술꾼"보다 부드러운 톤 후보 — `술친구`, `한잔러`, `딱한잔의 신` 등 (콘텐츠 작성 단계에서 확정)
- 음주 조롱 → 한국 회식·술자리 문화의 흔한 모먼트 풍자 (예: "딱 한 잔만이 다섯 잔이 되는", "내일 일찍 퇴근하면 됨")
- 결과 페이지 하단에 절주 메시지 + **보건복지부 알코올 상담 전화 1670-8268** 안내 (원작 의도 살리되 캠페인성 강화)

**구현 가이드:**
- 30개 필수 문항과 별개로 `questions.json`에 `hidden: true` 플래그 문항 1~2개 배치
- 트리거 조건: 음주·회식 관련 답변 패턴이 임계 점수 초과 시 (구체 임계값은 데이터 작성 후 튜닝)
- `scoring.js`에서 일반 27유형 매칭 결과 산출 후, 트리거 조건 충족 시 DRUNK로 오버라이드 (코드 흐름: `computeBaseType() → checkHiddenOverride()`)

**리스크 완화:**
